# 클래스

## 이 폴더의 목적

클래스가 왜 필요한지, 그리고 `constructor`가 어떤 역할을 하는지를  
**문제 → 해결** 구조의 코드로 확인한다.

## 폴더 구성

### 1.blueprint (설계도로서의 클래스)

- **1.problem.blueprint.js**  
  비슷한 구조의 객체를 리터럴로 반복 생성할 때 생기는 중복 문제를 보여준다.

- **2.solution.blueprint.js**  
  클래스를 사용해 공통 데이터 구조와 로직을 한 곳에 모아 해결한다.  
  → 클래스 = 객체를 찍어내는 설계도

### 2.constructor (초기 상태 설정)

- **1.problem.constructor.js**  
  초기 상태를 설정하지 않아 `undefined` 기반 연산으로 문제가 생기는 사례를 보여준다.

- **2.solution.constructor.js**  
  `constructor`에서 초기값을 설정해 인스턴스가 예측 가능한 상태로 시작하도록 한다.

## 핵심 포인트

- 클래스는 비슷한 객체를 구조적으로 생성하기 위한 설계도다.
- `constructor`는 객체의 초기 상태를 일관되게 설정하는 함수다.
- 올바른 초기화는 이후 로직의 예측 가능성을 높인다.

## 한 문장 정리

클래스는 구조를 통일하고, `constructor`는 안전한 출발선을 만든다.
