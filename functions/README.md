# 함수

## 이 폴더의 목적

함수의 핵심 역할(로직을 묶고, 필요할 때 호출해 실행 시점을 제어함)을 짧은 코드로 확인하기 위해 작성했다.

## 구성

- 1.declaration-call.js
  - 함수는 "선언"과 "호출"이 다르며, 호출될 때 실행된다.

- 2.return.js
  - `return`은 값을 반환하고 함수를 종료한다.
  - 조건에 따라 조기 종료(early return)할 수 있다.

- 3.parameters.js
  - default parameters: 인자가 `undefined`일 때만 기본값이 적용된다.
  - rest parameters: 남은 인자들이 배열로 모인다.

- 4.callbacks.js
  - 콜백 함수: 함수를 다른 함수의 인자로 전달해 실행을 위임할 수 있다.

## 핵심 포인트

- 함수는 호출 전까지 실행되지 않는다.
- `return`은 "값 반환 + 함수 종료"이다.
- default parameter는 `undefined`일 때만 적용된다.
- rest parameter는 남은 인자들을 배열로 받는다.
- 콜백을 사용하면 "무엇을 할지"를 함수 인자로 전달해 재사용성을 높일 수 있다.

## 한 문장 정리

함수는 로직을 묶고, 필요할 때 호출함으로써 실행 시점을 제어하는 단위다.
