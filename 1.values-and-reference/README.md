# 값과 참조

## 이 폴더의 목적

자바스크립트에서 값이 복사/공유되는 방식(원시 vs 객체)을 확인하고,  
 `const`의 의미와 불변성 유지의 중요성을 코드로 정리한다.

## 구성

- **1.primitive-vs-reference.js**
  - 원시 타입은 값이 복사되고, 객체 타입은 참조가 복사되어 같은 객체를 공유할 수 있다.

- **2.const.js**
  - `const`는 재할당은 막지만, 객체의 내부 값 변경은 막지 않는다.  
    (참조는 고정되지만 불변성을 보장하지는 않는다.)

- **3.immutability.js**
  - 객체를 직접 수정하면 사이드 이펙트가 생길 수 있다.
  - 불변성을 지키려면 기존 객체를 수정하지 말고 새 객체를 생성한다.

## 핵심 포인트

- 원시 타입: 값 복사, 객체 타입: 참조 복사(공유)
- `const` = 재할당 금지, 불변성 보장 X
- 불변성 유지: 기존 객체를 직접 수정하지 않고 새 객체로 업데이트

## 한 문장 정리

객체는 참조를 다루므로, 의도치 않은 공유/변경을 피하려면 불변성을 지키는 업데이트가 필요하다.
